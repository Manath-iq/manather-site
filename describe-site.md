Формат
Одностраничный лендинг (single-page, как у Raycast, CleanShot X, Linear, Arc, Screen Studio, Cron) — не мультистраничный сайт. Для open-source macOS-утилиты это стандарт: вся продажа идёт сверху вниз, один сильный CTA «Download», якорная навигация вместо переходов. Ориентируйся на эти сайты как референс стиля/подачи (тёмная тема, крупное видео-демо в hero, bento-grid фич, много живых GIF вместо статичных скриншотов).

Структура секций (сверху вниз)
1. Navbar (sticky)
Лого + название → Features / How it works / MCP / Download (якоря). Справа: GitHub-звёзды (live badge) + кнопка Download. Компактный, прозрачный, при скролле — фон.

2. Hero

Заголовок в духе README: «The native macOS home for everything you feed your AI»
Подзаголовок: одна фраза про боль → решение (references, skills, MCP, snippets, prompts → context pack одной кнопкой)
Два CTA: Download for macOS (основной, с иконкой Apple, версия/размер под кнопкой) + View on GitHub (вторичный, outline)
Подпись мелким текстом: «Apple Silicon · macOS 14+ · free & open source»
Главный визуал — не статичный скриншот, а зацикленное видео/GIF в рамке окна macOS: drag&drop картинки → она падает в masonry-сетку. Это самое конвертящее место на странице, стоит снять отдельно.
3. Полоса доверия (соцдоказательство)
Тонкая строка сразу под hero: GitHub stars, счётчик downloads (у тебя уже есть shield-бейджи в README — вынеси их сюда живыми), CI-бейдж «build passing», значок MIT. Дёшево добавляет доверия для open-source продукта.

4. Проблема → Решение (до/после)
Ты сам это предлагал — вот тут это уместнее всего. Разворот в два столбца:

До: скриншот бардака — папка Downloads, заметки в Notes, промпты в чате, skills разбросаны по ~/.claude, каждый новый проект начинается с нуля (можно нарисовать/сфоткать реальный беспорядок, не мокап)
После: тот же контент в аккуратной masonry-сетке Manather
Текст-мостик между ними — абзац «Why Manather» из README почти готов.
5. Как это работает — 3 шага
1×1 повтор структуры из README (Collect / Organize / Export), но каждый шаг — своя GIF-демка конкретного действия, а не иконка:

Collect → перетаскивание файла + вставка ссылки, скриншот сайта генерируется на глазах
Organize → добавление в несколько коллекций (chips), цветовой фильтр кликается
Export → правый клик → Export → выбор Claude Code → появляется папка с CLAUDE.md/context.md
6. Bento-grid фич
Сетка карточек разного размера (как у Raycast/Linear), у каждой — короткий GIF/скриншот, не абзацы текста:

Библиотека: 6 типов ассетов, masonry, цветовой фильтр
Коллекции: many-to-many, «New collection», карточки-веер
Boards — отдельная крупная карточка (это визуально самая эффектная фича — холст с заметками/стрелками/фигурами), стоит вставить видео с пан/зумом
AI-фичи: Generate variation + vision auto-tag (эффектно показать «до генерации / после»)
Мультибиблиотеки
Detail view с палитрой цветов
7. MCP-сервер — отдельная секция (это твой главный дифференциатор, не прячь в grid)

Заголовок: «Your AI agent can reach into your library directly»
Слева/справа: терминал с командой claude mcp add ... из README + анимация как агент ищет и забирает ассет, справа — сетка в приложении, куда что-то добавляется в реальном времени
Три буллета permissions/security: 127.0.0.1 only, токен, выключено по умолчанию — это снимает главное возражение «страшно давать агенту доступ к файлам»
8. Context Pack — что реально получает агент
Возьми блок из README с деревом файлов (CONTEXT.md, manifest.json, assets/, skills/, snippets/) и оформи как код-блок с подсветкой + рядом скриншот реального CLAUDE.md, сгенерированного тулом. Программистам это продаёт продукт лучше любых скриншотов UI.
